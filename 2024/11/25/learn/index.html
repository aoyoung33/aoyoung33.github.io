简单记录
花了两天的时间终于建好了一个属于自己的个人网站。中间因为配置不成功好几次想要放弃，但是又坚持下来。感谢互联网上发布教程的所有人，感谢大语言模型为我寻找可能的解决方法。

源头

建一个网站的动机来源很模糊，大概源头是最近举办的全国大学生职业规划大赛。我很认真地做了简历和PPT，可是院系阶段的面试答辩让我很受挫败。大家站在台上的范儿完全不一样，充满热情和掌控力，而我上台后，说话还磕磕绊绊，虽然我没有明显的紧张感，但是我就是没有好的状态和感染力。

失落之余我就看到了朋友圈里的一则，说是现在生成式人工智能让建个人网站轻而易举。我心想，建网站是一个多么技术性的工作？！自从我从教育技术转到教育学后，我对编程、计算机就绝缘了。也许这是一个机会，我心底永远留存着理科生的倔强。

开始动手

一旦开始搜索某个东西，大数据就会把我置身其中，逃不出相关的消息。持续了两天的信息冲击和理论输入，我决定动手开始了。

视频教程看了很多，文字教程也看了很多。大家各不相同，选择多样，步骤和方式也充满了发布者的个人特点和实际情况，很难找到一个万无一失和 100% 成功的方案。毕竟我自己这边的情况也是无法预测的。

顺利，失败，错误，成功，失败，错误······

反反复复，几度我想要放弃，怎么建一个静态网站都如此困难？可是我又不想放弃，不就是一个网站吗，我怎么就不行呢？

问题总结

其中有几个问题是我遇到后，找了很久很久原因和解决办法的，总结如下。

1.sudo很重要

终端使用 npm 安装某个东西的时候，需要加上 sudo
比如

sudo npm install -g cnpm --registry=https://registry.npmmirror.com —verbose

这是在安装 cnpm 时不能成功安装，找到的解决办法

sudo cnpm install --save hexo-deployer-git

npm install hexo-next-giscus --save

2.上传 hexo 到 github 失败需要使用 SSH连接 、选择 gh-pages

因为使用的是github pages的仓库来存储网页
先安装 hexo-deployer-git：
再在_config.yml 文件末尾的 Deployment 部分修改为

deploy:
  type: git
  repo: <在仓库的 SSH 部分复制>
  branch: gh-pages

注意，这里选择的分支是 gh-pages，所以要在（Repository Settings）中将默认分支设置为_config.yml 配置中的分支名称。
首先，general ：default branch选择 gh-pages
接着，在 pages页面下：
build and deployment 中选择 deploy from a branch
branch 中选择 gh-pages，root
参考文章： HEXO 系列教程 | 使用 GitHub 部署静态博客 HEXO | 小白向教程

3.更换主题不能太随意

当我顺利设置好一种好看的主题后，我竟然天真地以为我可以融合主题，从其他主题里面选择我喜欢和我需要的加到现在主题里。结果就是失败失败，问题很多。有一个主题中把网站中的导航栏设置了颜色，在每年的6月骄傲月的时候会自动变成彩虹。我想要，可是没能实现。

可能这些还得再探索探索。

4.本地预览没问题，部署到github上时在浏览就无法正常显示

这个问题我现在也不知道如何解决，我选择重来几遍。

5.categories 和 tags的表示

如果一篇文章要有多个 tags 和 categories 时，需要用[]括起来，使用,和空格隔开
比如

tags: [网站, 自学, GitHub]
categories: [学习]

单词不要写错了！（我真的就是犯这种小错误）
还需要新建

hexo new page tags

并在生成的 md文件中
在—和—中间替换为：type: tags

6.评论设置要去单独安装评论系统

比如 giscus，按照要求去新建仓库、开放评论功能、安装、填写、修改
官网页面的空格里是填写【用户名/仓库名（新建的评论仓库）】，不是用户名或者仓库名

7.背景流星效果为防止一进入主页就突然变白，需要在 layout.ejs中把位置改为-2

  <canvas
    id="background"
    style="position: fixed; top: 0; left: 0; width: 100vw; height: 100vh; pointer-events: none; z-index: -2"
></canvas>
  <script src="/js/background.min.js"></script>

还有很多需要探索的，想要记录我的故事。
